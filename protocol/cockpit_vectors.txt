# Golden wire vectors for the cockpit protocol.
# Spec: protocol/cockpit_protocol.md (sections 2, 3, 8).
# Consumed by BOTH test suites; a change here fails whichever side disagrees:
#   firmware/common/tests/test_codec.cpp   (airframe: parse in, format out)
#   pilot/tests/test_wire_vectors.py       (pilot:    format out, parse in)
# Lines starting with '#' and blank lines are ignored. Sections in [brackets].
# Fields are separated by '|'. Trailing/leading spaces in fields are DATA.
#
# [parse]  airframe input classification + tokens
#   line|request|tok0|tok1|...
#   line|relay|payload
#   line|ignore
# [format] airframe output formatting
#   ok|verb|fields|expected           (fields may be empty)
#   err|verb|reason|detail|expected   (detail may be empty)
#   state|from|to|expected
#   fault|code|expected
# [request] pilot request formatting (op + params -> wire line)
#   ping|ping
#   drive|linear|angular|expected
# [downlink] pilot parsing of airframe lines
#   line|ok|verb|k=v ...              (fields joined by spaces, may be empty)
#   line|err|verb|reason|detail       (detail may be empty)
#   line|state|from|to
#   line|fault|code
#   line|log|text
#   line|relay|payload
#   line|skip
